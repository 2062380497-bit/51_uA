# 电路功能说明

本文基于当前原理图截图、网表和 BOM 做整理，目的是把各部分电路的作用说清楚，方便后续改版和调试。由于硬件不在现场，以下结论以现有连接关系为准，个别脚位仍建议上板后用万用表和示波器复核。

## 1. 整体功能

这块板子的核心目标是做一个由 MCU 控制的电流刺激输出装置。系统大致分为以下几部分：

- USB 供电和充电管理
- 电池供电与电源切换
- MCU 主控
- 升压电路
- 输出电极驱动与电流控制
- 放电/保护电路
- 蜂鸣器、按键、指示灯和串口通信

其中最关键的是升压、电流输出和输出保护三部分，因为它们直接决定刺激波形和人体安全性。

## 2. USB 供电与充电管理

相关器件主要是 USB-C 接口和 U6（TP4055）。

作用如下：

- USB-C 负责引入 5V 电源。
- TP4055 负责给单节锂电池充电。
- 板上存在 USB 供电状态下的逻辑检测，用于判断当前是否插入 USB。

这类设计通常会让系统在 USB 供电时进入安全模式，例如禁止刺激输出，避免边充电边输出到人体。

## 3. 电池与主电源

从网表看，H2 一侧标注为 3.7V 电源输入，说明主系统很可能由单节锂电池供电。

相关节点包括：

- POWER：主电源节点
- GND：系统地
- C105、C106、C82 等：电源去耦和储能电容

这一块的作用是给 MCU、升压级和输出级提供基础电源，并在脉冲刺激时提供瞬时电流缓冲。

## 4. 升压电路

升压部分由 L1、Q42、D3、C106、相关电阻和 MCU 的 PWM 控制共同组成。

### 4.1 作用

它的作用是把电池电压升高到更高电平，供后续输出级使用。由于单节锂电池只有约 3.0V 到 4.2V，直接做刺激输出通常不够，所以需要升压。

### 4.2 工作方式

从连接方式看，这是一类 MCU 驱动的开环或半开环升压结构，而不是典型专用升压芯片闭环控制。

一般流程是：

1. MCU 输出 PWM。
2. Q42 作为开关管控制电感 L1 充放能。
3. D3 对升压节点进行整流/续流。
4. C106 对升压输出进行滤波和储能。

### 4.3 说明

这类电路的输出电压范围通常取决于占空比、频率和负载，不是固定值。轻载时电压会升得更高，重载时则会下降。

## 5. 升压电压检测与状态判断

相关网络里有 BOOST、ADC1、R179、R182、R183 等。

### 5.1 可能作用

这一块很可能不是单纯的“ADC 采样升压电压”，而是一个复用节点：

- 在某些状态下用于检测升压电压或电源状态
- 在插入 USB 后用于做电源存在判断或逻辑切换

从当前连接关系看，ADC1 通过电阻同时连接到 +5V 和 BOOST 相关节点，这种结构不太像标准分压采样，更像是状态检测/复用输入。

### 5.2 设计含义

如果它确实是 MCU 的输入脚，那么 100k 级电阻通常用于：

- 给输入一个弱偏置，避免悬空
- 在不同供电状态下形成可识别的电平
- 限制异常情况下的输入电流

## 6. 输出电极与电流控制

输出端为 P+、P-，这是直接接人体的端口。

相关器件包括 Q36、Q37、Q38、Q39、部分 33k 和 100k 电阻、以及 ADC 反馈网络。

### 6.1 作用

这部分负责把升压后的能量变成受控的刺激电流，并通过不同晶体管组合实现通断、方向切换或电流路径控制。

### 6.2 结构特征

从器件和网络命名看，这一块更像是：

- 由 MCU 控制的开关阵列
- 通过不同管子实现输出极性的切换或路径选择
- 同时将部分输出节点回送到 MCU 进行检测

### 6.3 注意点

这类接人体输出必须重点关注：

- 是否有硬件限流
- 是否可能出现直流分量
- 是否在 MCU 异常时保持默认关断

## 7. CONTROL1 / CONTROL2 / CONTROL3

这几个控制脚是 MCU 到输出级或辅助电路的控制信号。

### CONTROL1 / CONTROL2

它们看起来更像是输出级的分支控制信号，用来驱动某些晶体管，决定输出路径、波形或者相位。

### CONTROL3

CONTROL3 连接到 Q41，Q41 又和 POWER 节点、C105 相关联。根据这个连接关系，它大概率不是输出刺激本身，而是：

- 电容余电放电控制
- 电源节点泄放
- 关机或切换状态下的残压处理

换句话说，它更像保护和复位用途，不是直接刺激输出脚。

## 8. 电容余量放电

Q41 周边的连接显示，POWER 节点上有电容储能，而 Q41 负责在某些时机把残余电荷快速放掉。

这部分的作用是：

- 防止电源关断后仍有残压
- 避免再次上电时状态异常
- 提高输出关断的安全性

对于刺激类设备，这一块很重要，因为残压会影响波形可重复性，也会影响停机后的安全状态。

## 9. 蜂鸣器、按键、LED

这部分是典型的人机交互电路。

### 蜂鸣器

- 用于提示开机、动作完成、报警等状态

### 按键 SW2 / SW4

- 用于控制模式切换、启动/停止或参数选择

### LED3 / LED4 / LED5 / LED7

- 用于显示电源状态、工作状态、充电状态或异常状态

这一部分和刺激功能本身关系不大，但对调试体验很重要。

## 10. USB 转串口

U4 是 CH340E，说明板上预留了 USB 转串口通信功能。

作用如下：

- 下载或调试 MCU 串口数据
- 打印参数、状态和故障信息
- 方便开发阶段联机调试

## 11. 目前这版图纸的初步结论

基于现有资料，我对各部分的判断是：

- 升压部分是合理的，属于 MCU 驱动的开关升压结构
- CONTROL3 很可能是放电或关断控制，不像刺激输出主通道
- ADC1 很可能是复用输入，不一定只是单纯的升压采样
- USB 在位后关闭 PWM 的设计思路是合理的
- 接人体的输出部分还需要重点确认是否有硬件限流和直流分量抑制

## 12. 建议的下一步

如果后续还要改版，建议优先确认以下三件事：

1. ADC1 的真实用途，是采样、检测还是 USB 在位判断。
2. P+ / P- 输出是否具备硬件限流和失控保护。
3. CONTROL3 是否确实只用于放电，不参与刺激脉冲本身。
